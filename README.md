# 호텔 예약 사이트 만들기

# Travel Maker

- 개발 절차
    1. 아이디어와 계획 수립:
        - 웹사이트의 목적과 기능을 정의하고, 타겟 시장을 식별합니다.
    2. 웹사이트 디자인 및 개발:
        - 웹사이트의 디자인과 사용자 경험을 설계합니다.
        - 웹사이트 개발을 위해 프로그래밍 언어와 기술 스택을 선택합니다.
        - 호텔 정보, 예약 시스템, 결제 시스템 등의 기능을 개발합니다.
    3. 호텔 데이터베이스 구축:
        - 호텔 정보와 가격 데이터를 수집하고 데이터베이스를 구축합니다.
        - 호텔 상세 정보, 가격, 예약 가능 여부 등을 저장합니다.
        - API 활용:
            - 호텔 예약 사이트나 호텔 제공 업체들은 API를 통해 데이터를 제공하기도 합니다. 대표적인 예로는 Expedia, Booking.com, Airbnb 등이 있습니다.
            - 장점:
                - 공식 API를 사용하면 데이터의 정확성과 신뢰성이 높습니다.
                - 데이터 업데이트는 자동으로 이루어집니다.
                - 법적 제약 사항과 서버 부하에 대한 걱정이 줄어듭니다.
            - 단점:
                - API 사용에는 사용료가 있을 수 있으며, 초과 사용에 대한 비용도 고려해야 합니다.
                - API 제공자의 제한된 기능과 데이터 포맷을 준수해야 합니다.
                - 어떤 호텔 예약 업체들은 API 액세스에 대한 제한을 둘 수 있습니다.
    4. 예약 및 결제 시스템 구현:
        - 사용자가 호텔을 검색하고 예약할 수 있는 기능을 개발합니다.
        - 안전한 결제 시스템을 통합하여 예약을 완료할 수 있게 합니다.
    5. 검색 및 필터링 기능 추가:
        - 사용자가 호텔을 쉽게 검색하고 필터링할 수 있는 기능을 구현합니다.
        - 가격, 위치, 별점, 편의 시설 등 다양한 옵션을 제공합니다.
    6. 사용자 계정 및 프로필 관리:
        - 사용자가 계정을 만들고 프로필을 관리할 수 있는 기능을 추가합니다.
        - 이력, 예약 기록 및 즐겨찾기 호텔을 저장할 수 있도록 합니다.
    7. 리뷰 및 평가 시스템:
        - 사용자가 호텔을 평가하고 리뷰를 남길 수 있는 기능을 제공합니다.
        - 다른 사용자에게 호텔 선택을 도와주는 리뷰 시스템을 구축합니다.

# 1. 아이디어와 계획 수립

## 아이디어

1. 사용자가 대략적인 일정을 입력하면 적절한 숙소 위치 및 숙소 리스트 추천 ( 구현하기 매우 어려울 듯)
2. 호텔과 비행기 예약을 같이 장바구니 시스템
3. 호텔 왕복, 비행기 예약을 하나의 묶음으로 저장해 다른 방안과 가격 및 비행기 출발 시간 등 비교
4. 호텔 모듈화로 구현했을때 유튜브 영상이 있는지 표시?

## 1. 무엇을 하는 서비스인가?

> 비행기와 호텔 에약 정보를 제공하고 이 둘을 패키지로 묶어 비교가 용이하도록 하는 서비스
> 

## 2. 서비스의 간단한 흐름

1. 자신의 여행 정보 입력
2. 원하는 호텔 비행기를 선택
3. 결제 혹은 패키지로 만들어 저장
4. 또 다른 비교할 패키지를 선택
5. 비교
6. 결제

## 3. 사용자의 이용 흐름

```flow
user 방문
여행 출발지, 여행 목적지, 출발 도착일, 인원 입력 후 검색
loop start
비행기편 가격 확인 후 묶음에 추가
숙소 가격 확인후 묶음에 추가
비교 or 결제
화면에 (다른 일정으로 비교해보시겠습니까?) y/n
if y: 
	  비교 목록에 묶음 추가
	  go start
else : 
		결제
    end loop 
loop end
결과를 호텔측 페이지에 보냄
```
